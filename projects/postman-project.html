Postman API Testing & Documentation

March 2025 API Testing & Documentation

Project Overview

As a Developer Advocate with a background in testing, I created a comprehensive API testing project using Postman that not only validates API functionality but also serves as interactive documentation for developers.

Technologies Used

  • Postman - Primary API testing tool
  • JSON - Test data creation and response validation
  • JavaScript - Test scripting
  • Newman - Command-line execution
  • GitHub Actions - CI/CD integration
View GitHub Repository Live Documentation

Project Details

Challenge

The challenge was to create a comprehensive API testing suite that not only validated API functionality but also served as interactive documentation for developers. The goal was to demonstrate how effective test suites can double as powerful developer resources.

Approach

I approached this project with a developer-first mindset, focusing on creating tests that were not only thorough but also educational. Each test includes:

  • Clear and descriptive test names that communicate purpose
  • Pre-request scripts that explain what's happening before each request
  • Test scripts with detailed comments explaining validation logic
  • Environment variables to demonstrate flexible testing across environments
  • Examples showcasing both successful and error scenarios

Key Features

1. Request Validation

Created comprehensive test assertions to validate:

  • Status codes for both successful and error responses
  • Response time requirements
  • JSON schema validation
  • Content validation using JavaScript

2. Dynamic Environment Variables

Implemented flexible environment handling to demonstrate:

  • Testing across development, staging, and production environments
  • Automatic token handling and authentication
  • Data persistence between requests (saving values from one response to use in subsequent requests)

3. Automation with Newman

Set up command-line automation to:

  • Execute tests via the command line for CI/CD integration
  • Generate detailed HTML reports after test execution
  • Provide clear pass/fail results for each test case

4. Interactive Documentation

Published the collection as interactive documentation that:

  • Provides a user-friendly interface for exploring API endpoints
  • Includes examples of request and response bodies
  • Offers detailed descriptions of all parameters and options
  • Allows developers to try out API calls directly from the documentation

Developer-Focused Documentation

A key aspect of this project was transforming technical test cases into valuable developer resources. The documentation includes:

  • Getting started guides for API onboarding
  • Detailed parameter explanations
  • Common use cases and examples
  • Troubleshooting guides for common errors
  • Step-by-step tutorials for complex workflows

Results & Impact

The project demonstrates how well-designed API testing can provide multiple benefits:

  • Reduced onboarding time for new developers using the API
  • Increased test coverage ensuring API reliability
  • Automated validation in CI/CD pipelines
  • Improved developer experience through clear, interactive documentation
  • Testing as living documentation that evolves with the API

Code Snippets

Pre-request Script Example

// Set up test data dynamically
const timestamp = new Date().getTime();
const uniqueId = `test-user-${timestamp}`;

// Store in environment variables
pm.environment.set("uniqueId", uniqueId);
pm.environment.set("userEmail", `${uniqueId}@example.com`);

console.log("Test data prepared:", uniqueId);

Test Script Example

// Basic response validation
pm.test("Status code is 200 OK", function () {
    pm.response.to.have.status(200);
});

// Response time validation
pm.test("Response time is acceptable", function () {
    pm.expect(pm.response.responseTime).to.be.below(500);
});

// Schema validation
const schema = {
    "type": "object",
    "required": ["id", "name", "email"],
    "properties": {
        "id": { "type": "string" },
        "name": { "type": "string" },
        "email": { "type": "string", "format": "email" }
    }
};

pm.test("Response schema is valid", function () {
    const jsonData = pm.response.json();
    pm.expect(tv4.validate(jsonData, schema)).to.be.true;
});

// Business logic validation
pm.test("User email matches expected format", function () {
    const jsonData = pm.response.json();
    const expectedEmail = pm.environment.get("userEmail");
    pm.expect(jsonData.email).to.eql(expectedEmail);
});
